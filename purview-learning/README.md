# Microsoft Purview — Learning Curriculum

Expert-level training on Microsoft Purview with focus on Power Platform governance.
Each module distinguishes **industry standard** concepts from **Microsoft-specific** implementations.

## Modules

| # | Module | Focus | Status |
|---|--------|-------|--------|
| 01 | [Foundations](docs/md/01-foundations.md) | Governance landscape, Purview evolution, portal navigation, licensing | Draft |
| 02 | Unified Audit Log | Audit concepts, UAL deep dive, Power Platform audit entries | Planned |
| 03 | Information Protection | Data classification, sensitivity labels, auto-labeling | Planned |
| 04 | Data Loss Prevention | Purview DLP vs Power Platform DLP, policies, enforcement | Planned |
| 05 | AI Governance | AI Hub, DSPM for AI, Copilot monitoring, NIST AI RMF | Planned |
| 06 | eDiscovery & Legal Hold | Content search, eDiscovery Standard/Premium, legal holds | Planned |

| 08 | Data Governance | Data Map, catalog, lineage, scanning, classification | Planned |
| 09 | Power Platform Deep Dive | Copilot Studio compliance, custom connectors, managed environments, DLP | Planned |
| 10 | Applied — Demo Narrative | Purview + DSS legal case demo, GCC realities, what to show customers | Planned |

## Prerequisites
- Azure subscription with Purview access
- Microsoft 365 E5 or equivalent compliance licensing
- Power Platform environment (Copilot Studio)
- Familiarity with: Semantic Kernel, MCP, APIM, managed identity patterns

## How to Use
Work through modules in order. Each module builds on the previous.
Modules are in `docs/md/`. PDFs, HTML, and slides will be generated as needed.
